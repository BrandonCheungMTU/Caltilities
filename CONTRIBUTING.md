## Getting Started

### Prerequisites

- **Python 3.9 or higher** - Our core language
- **Git** - For version control
- **Basic understanding of**:
  - Python programming
  - Object-oriented programming concepts
  - (Optional) QML or declarative UI frameworks

### What to Contribute

We welcome contributions of all sizes:

| Type | Description | Good for Beginners? |
|------|-------------|---------------------|
| 🐛 Bug fixes | Fixing reported issues | ✅ Yes |
| 📝 Documentation | Improving README, guides, docstrings | ✅ Yes |
| ✨ New features | Adding new mathematical functions or UI components | 🔶 Medium |
| 🎨 UI improvements | Enhancing QML interface, animations, themes | 🔶 Medium |
| 🧪 Tests | Writing unit tests for existing code | ✅ Yes |
| 🔧 Performance | Optimizing mathematical computations | 🔴 Advanced |
| 🌐 Localization | Translating the interface to other languages | ✅ Yes |

## How to Contribute

### Step 1: Find Something to Work On

- Browse [open issues](https://github.com/[your-username]/caltilities/issues)
- Look for labels like:
  - `good-first-issue` - Perfect for beginners
  - `help-wanted` - Actively seeking contributions
  - `bug` - Bug fixes needed
  - `enhancement` - Feature improvements

**Don't see what you want?** Create a new issue describing what you'd like to work on.

### Step 2: Fork and Clone

```bash
# Fork the repository on GitHub, then clone your fork
git clone https://github.com/your-username/caltilities.git
cd caltilities

# Add the original repository as upstream
git remote add upstream https://github.com/original-owner/caltilities.git
